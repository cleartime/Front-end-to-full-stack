# 前端转全栈路线

## 1. 前端优势不要丢

前端同学转全栈，最大的优势不是「少学一点后端」，而是你已经懂用户路径、状态变化、性能体验和工程协作。转型时要把这些能力映射到服务端：

- 组件边界 -> 服务分层和模块边界。
- 状态管理 -> 领域状态、事务状态、缓存状态。
- 首屏性能 -> API 延迟、数据库索引、缓存策略。
- 用户体验 -> 错误码、重试、降级、流式响应。

## 2. 必学能力

- Node.js 基础：事件循环、异步 I/O、错误处理、进程模型。
- HTTP 与 REST：状态码、幂等性、缓存头、分页、版本化。
- 数据库：表设计、外键、索引、事务、慢查询、N+1。
- 鉴权安全：Cookie、Session、JWT、RBAC、CSRF、XSS、CORS、限流。
- 工程质量：TypeScript、目录分层、日志、测试、环境变量。
- 部署运维：Docker、CI/CD、健康检查、日志、监控、回滚。
- AI 工程：流式输出、prompt 注入、工具权限、输出校验、成本控制、评测。

## 3. 学习顺序

1. 用 Next.js Route Handler 写 REST API，先掌握请求到响应。
2. 加 Prisma 和数据库，练 CRUD、分页、事务。
3. 加登录鉴权，理解 Cookie/JWT/RBAC 的安全边界。
4. 加日志、限流、测试，补工程化能力。
5. 加 AI streaming 或队列任务，练现代业务常见集成。
6. 最后准备系统设计表达：读写路径、瓶颈、缓存、扩展、可观测性。

## 4. 每周训练法

- 周一到周三：写一个后端能力点，例如事务、鉴权或限流。
- 周四：写测试和 README，记录为什么这样做。
- 周五：用面试口吻复述「需求 -> 设计 -> 代码 -> 权衡」。
- 周末：做一次重构或扩展，例如换数据库、加 Redis、接真实 AI provider。
